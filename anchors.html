Anchors, and why they're useful.

A quick and terribly written document by Ducky.




Anchors are like pre-made bookmarks that you can attach to almost any element. You can set a name for each of these "bookmarks", and it will jump down to them. For example, I'm going to set a bunch of new lines so you have to scroll to the next thing, but who wants to do that? Just click here.

































Bang! You just jumped to here!

Now wasn't that just useful?




People use anchors all the time, whether it be to organize a long document, or just plainly because they don't want to make seperate pages. Wanna learn how to use them? Well, why don't we use one to take you there? Let's go!

































Here we are! Now let's get to coding an anchor!




First, find an element that you want to make an anchor for. Usually you will use some text element, so to do this, put <OriginalElement><a name="WhateverYouWantToNameTheAnchor">TextFromOriginalElement</a></OriginalElement>. This will keep the same appearance of the element, but set an anchor on the element, so if you link to it, it knows where to go.

Now you need someway to get to it. Type <a href="#WhateverYouWantToNameTheAnchor">LinkText<a/a>. Notice that you need a # in the link for your website to recognize it as an anchor, but you don't need one when setting the anchor in our earlier step. This entire example creates a anchor named "WhateverYouWantToNameTheAnchor", and has a link that jumps to it with the text "LinkText". Pretty easy, huh?
